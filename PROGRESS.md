## Як працює AI цикл

1. Користувач пише текст або кидає фото
2. `client.py` буферизує 3 сек (збирає кілька фото в одну пачку)
3. `ai.py chat()` — збирає контент (фото + текст), викликає Claude
4. Claude відповідає текстом + JSON-командами в кінці
5. `_parse_multi_actions()` витягує всі JSON блоки
6. `_execute_action()` виконує кожну дію
7. JSON блоки вирізаються з відповіді, юзер бачить тільки текст

**Доступні дії Claude:**

- `add_to_shopping` / `remove_from_shopping` / `clear_shopping`
- `update_inventory` (статуси: є / мало / нема)
- `add_to_freezer` / `remove_from_freezer`
- `save_cooking_style` — запам'ятати кулінарний підхід
- `save_recipe` / `remove_recipe` — рецепти
- `no_action`

Системний промпт `build_system()` — динамічний, збирається при кожному запиті.
Має `cache_control: ephemeral` для економії токенів.

## Що зроблено (Фаза 2) ✅

### Кулінарний мозок (core/kitchen.py)
- Новий модуль: рецепти, кулінарні підходи, purchase history
- `save_cooking_style` — Мег запам'ятовує як сім'я готує
- `save_recipe` / `remove_recipe` — збереження рецептів з тегами
- Все вбудовано в системний промпт через `format_for_prompt()`

### Профілі сім'ї (data/family.json) — заповнено
- Саша — любить гостре, сирий цибуля, всі кухні. Не любить чорнослив, суху телятину, жирну свинину
- Ксюша — любить сир, макарони, помідори, десерти, гостре, шашлик
- Сашулік (1.5 року) — їсть те саме що дорослі. Без цукру, сіль у межах

### Кулінарні підходи (data/recipes.json) — заповнено
- Безлактозна молочка
- М'ясо завжди маринуємо
- Система Souper Cubes: малий ~1cup + великий ~2cup
- Після заморозки — харчова плівка + наліпка з назвою і датою
- Готуємо заготовками під розморозку (пайплайн СК)
- Любимо гостре, всі кухні світу, сир скрізь

### Purchase history (data/purchase_history.json) — заповнено
- 12 категорій: молочне, хліб, фрукти/овочі, м'ясо, делікатеси, соуси, дитяче, крупи, снеки, солодощі, консерви, побутове

### Морозилка (data/freezer.json) — повністю заповнено
- 2 морозилки, 10 ящиків (1-7 перша, 8-10 друга)
- Нумерація локацій: `ящик N`
- Одиниці: великий (~2cup СК), малий (~1cup СК), пакет, упаковка, шт
- Заповнено вручну через фото з наліпками — дати з наліпок збережено
- Потребує дрібних правок у наступній сесії

### Двокроковий режим для фото
- При отриманні фото — Мег показує список що знайшла (не зберігає)
- Юзер перевіряє, виправляє і надсилає назад зі словом "зберегти"
- Тільки після підтвердження — виконує JSON команди

### Формат виводу морозилки
- Групування по ящиках, сортування за назвою
- Одиниці з правильним відмінюванням: великий/великих, малий/малих
